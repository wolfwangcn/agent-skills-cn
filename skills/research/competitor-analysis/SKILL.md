# 竞品分析 (Competitor Analysis)

基于顶级咨询公司方法论（麦肯锡 3C/7S、BCG 矩阵、Porter's Five Forces、Blue Ocean 战略画布），进行专业级竞品对比分析。

---

## 方法论工具箱

### 根据场景选择框架

| 分析目的 | 推荐框架 | 核心问题 |
|---------|---------|---------|
| 行业竞争格局 | **Porter's Five Forces** | 这个行业赚钱容易吗？ |
| 单一竞品 SWOT | **SWOT + TOWS** | 竞品的软肋在哪？ |
| 产品差异化 | **Value Proposition Canvas** | 我们怎么不同？ |
| 市场定位 | **Blue Ocean 战略画布** | 哪里没人竞争？ |
| 投资组合 | **BCG 矩阵** | 哪个产品值得投？ |
| 增长对比 | **AARRR 漏斗** | 增长引擎差异在哪？ |
| 需求优先级 | **Kano 模型** | 做什么功能赢？ |

### Porter's Five Forces（行业分析）

```
         新进入者威胁
              ↓
供应商议价力 → 同业竞争 ← 买家议价力
              ↑
         替代品威胁
```

每个力评级：高/中/低 + 具体原因

### 麦肯锡 3C 模型

- **Company**（自身）：核心竞争力、资源、短板
- **Customer**（客户）：需求、痛点、决策因素、转换成本
- **Competitor**（竞品）：战略、能力、反应模式

### BCG 矩阵

```
市场增长率
  高 │ ★ 明星    │ ❓ 问号
     │（投入维持）│（决策：投或退）
  ───┼───────────┼──────────
  低 │ 🐄 金牛   │ 🐕 瘦狗
     │（收割利润）│（考虑退出）
     └───────────┴──────────
       高          低
       相对市场份额
```

### Value Proposition Canvas

```
┌─────────────────┐     ┌─────────────────┐
│ Customer Profile│     │ Value Map       │
│                 │     │                 │
│ • Jobs (任务)   │ ←→  │ • Products      │
│ • Pains (痛点)  │ ←→  │ • Pain Relievers│
│ • Gains (收益)  │ ←→  │ • Gain Creators │
└─────────────────┘     └─────────────────┘
```

## 指令

对用户指定的产品进行专业竞品分析。先确认分析目标和范围，再系统采集数据，最后输出带明确推荐的决策报告。

### 第一步：明确分析目标

1. **分析谁 vs 谁？** 你的产品 vs 哪些竞品（2-4 个最佳）
2. **分析目的？**
   - 产品选型（用户视角：选哪个）
   - 竞争策略（创业者视角：怎么赢）
   - 投资判断（投资者视角：投谁）
   - 市场研究（分析师视角：格局如何）
3. **核心决策维度？** 功能/价格/体验/技术/市场/团队
4. **信息深度？** 快速对比(30min) / 标准分析 / 深度报告

### 第三步：分析框架

根据分析目的选择适合的框架（见方法论工具箱），生成：

1. **定量对比**：功能矩阵 + 评分
2. **定性分析**：策略/定位/护城河
3. **趋势判断**：谁在变强/变弱

## 规则

- **结论放最前面** — 忙人只看前 30 秒
- **有数据就用数据** — "DAU 约 50 万"比"用户量很大"有用
- **标注数据来源** — 每个关键数据说明从哪来
- **区分事实和推测** — 确认的标事实，推断的标 [推测]
- **必须有明确推荐** — 不能"各有千秋"结尾
- **考虑时间维度** — 不只看现在，也看趋势（谁在变强/变弱）
- **分析完主动问** — "需要深入看哪个方面？要不要我出一份竞争策略？"
