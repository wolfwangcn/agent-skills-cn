# 竞品分析 (Competitor Analysis)

对指定产品进行结构化竞品分析，输出决策级对比报告。

## 指令

帮用户做竞品分析。先明确分析目标和维度，再系统收集信息，最后给出带理由的推荐。

## 分析前确认

1. **分析谁？** 你的产品 vs 哪些竞品
2. **核心维度？** 功能/价格/体验/技术/市场
3. **决策场景？** 选型/报告/融资材料/了解市场

## 信息收集维度

- 产品定位和目标用户
- 核心功能和差异化
- 定价模型
- 用户评价（App Store、ProductHunt、V2EX、知乎）
- 团队/融资（如果相关）
- 技术架构（如果可知）
- 市场份额/增长趋势

> 日期：YYYY-MM-DD

## 一句话结论
[直接说哪个好，为什么]

## 产品概览
| | A | B | C |
|---|---|---|---|
| 一句话定位 | | | |
| 目标用户 | | | |
| 核心卖点 | | | |

| 成立时间 | | | |

## 功能矩阵

|------|---|---|---|
| 功能1 | ✅ | ❌ | ⚠️(部分) |
| 功能2 | ✅ | ✅ | ❌ |

## 深度对比

### [产品A]
- 👍 核心优势：
- 👎 明显短板：
- 💰 定价分析：
- 📈 趋势：在变好还是变差？

### [产品B]
...

[个人观点 + 理由，不骑墙]

## 规则

- 结论放最前面 — 忙的人只看第一段
- 区分事实和观点，标注来源
- 不面面俱到 — 聚焦用户决策需要的维度
- 发现信息矛盾时标注 [数据冲突，待验证]
- 必须有明确推荐，不能"各有千秋"结尾
- 分析完问："要我深入看哪个方面？"
