# 竞品分析 (Competitor Analysis)

## 描述
对指定产品/项目进行结构化竞品分析，输出清晰的对比报告。

## 触发条件
用户提到"竞品"、"对比"、"XX 和 YY 哪个好"、"市场上有什么"

## 流程

### 第一步：明确目标
1. 分析谁？（你的产品 vs 竞品）
2. 分析维度？（功能/价格/体验/技术/市场）
3. 用途？（决策/报告/融资材料）

### 第二步：信息收集
- 产品官网和文档
- 用户评价（App Store、ProductHunt、V2EX）
- 定价方案
- 融资/团队信息
- 社交媒体声量

### 第三步：输出格式

```markdown
# 竞品分析：[产品A] vs [产品B] vs [产品C]

## 一句话结论
[直接说推荐哪个/哪个更好]

## 产品概览
| | 产品A | 产品B | 产品C |
|---|---|---|---|
| 定位 | | | |
| 目标用户 | | | |
| 核心卖点 | | | |
| 价格 | | | |

## 功能对比
| 功能 | A | B | C |
|------|---|---|---|
| 功能1 | ✅/❌ | ✅/❌ | ✅/❌ |

## 优劣势分析

### 产品A
- 👍 优势：
- 👎 劣势：

### 产品B
- 👍 优势：
- 👎 劣势：

## 建议
如果你是 [场景A] → 选 [X]
如果你是 [场景B] → 选 [Y]
```

## 规则
- 结论放最前面
- 信息要可验证（标注来源）
- 区分事实和观点
- 不要面面俱到——聚焦用户关心的维度
