# 技术调研 (Tech Research)

系统化调研某个技术方向，输出结构化报告和明确推荐。

## 指令

对用户指定的主题进行全面调研，收集数据，做出对比分析，给出明确推荐。不说"各有优劣看你需求"的废话。

## 调研前确认

1. **调研什么？** 技术/工具/框架/趋势
2. **目的？** 选型决策 / 学习了解 / 写报告
3. **约束？** 预算、团队规模、现有技术栈
4. **深度？** 快速概览(5min) / 标准对比(详细) / 深度分析(含源码级)

## 信息来源

- GitHub：stars、活跃度、issue 响应速度
- 官方文档：成熟度、API 设计
- 社区：Reddit、HN、V2EX、知乎讨论
- Benchmark：性能数据（标注来源和条件）
- 实际使用：如果能快速跑 demo，就跑一个

> 日期：YYYY-MM-DD | 信息时效：截至本月

[直接说推荐什么，为什么]

## 候选方案

|------|---|---|---|

## 详细分析

## 规则

- **有数据说数据，有观点标"个人观点"**
- 优缺点同样重要 — 不能只列优点
- 必须给明确推荐（"如果是我，我选 X"）
- 标注信息时效性
- 中文生态/社区支持是重要维度
- 如果信息不确定，标注 [待验证]
- 调研完问用户："需要我深入看哪个方案？"
