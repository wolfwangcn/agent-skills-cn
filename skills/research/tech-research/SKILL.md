# 技术调研 (Tech Research)

## 描述
系统化调研某个技术方向、工具或趋势，输出结构化报告。

## 触发条件
用户提到"调研"、"对比"、"看看最近有什么"、"分析一下"

## 流程

### 第一步：明确范围
1. 调研什么？（技术/工具/框架/趋势）
2. 目的是什么？（选型/学习/投资/写文章）
3. 深度要求？（概览 / 详细对比 / 深入分析）
4. 时间范围？（最近一周/一月/一年）

### 第二步：信息收集
- GitHub trending / stars 数据
- 官方文档和 changelog
- 社区讨论（Reddit、HN、V2EX）
- 性能/benchmark 数据（如果有）

### 第三步：输出报告

```markdown
# [主题] 技术调研报告

日期：YYYY-MM-DD

## 概述
一段话说明调研范围和结论

## 核心发现
1. [最重要的发现]
2. [次重要的发现]
3. [第三重要的发现]

## 详细对比（如适用）
| 维度 | A | B | C |
|------|---|---|---|
| 特点 | | | |
| 优势 | | | |
| 劣势 | | | |
| 适用 | | | |

## 推荐
根据你的场景，建议 [X]，原因是 [Y]

## 风险/注意
- [需要注意的坑]

## 参考链接
- [来源]
```

## 规则
- 有数据说数据，没数据说观点并标注
- 不要只列优点——缺点和风险同样重要
- 给明确推荐，不要"各有优劣看你需求"的废话
- 注明信息时效性（"截至 2024 年 X 月"）
