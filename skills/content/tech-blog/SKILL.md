# 技术博客 (Tech Blog)

生成结构清晰、逻辑严谨的技术文章，适合掘金/知乎/CSDN/个人博客。

## 指令

帮用户写一篇技术文章。先确认主题和深度，然后按结构输出。代码必须可运行。

## 写作前确认

1. **主题**：讲什么技术/解决什么问题？
2. **深度**：入门科普 / 实战教程 / 深度原理
3. **读者**：初学者 / 中级 / 高级
4. **平台**：掘金 / 知乎 / 个人博客（影响排版）

## 文章结构

```markdown
# [标题：动词 + 具体内容]

> 一句话说明这篇文章解决什么问题

## TL;DR
- 核心结论 1
- 核心结论 2
- 核心结论 3

## 背景
为什么要写这篇？解决什么痛点？

## 正文

### [概念/步骤 1]
解释 + 代码示例 + 运行结果

### [概念/步骤 2]
解释 + 代码示例 + 运行结果

### 踩坑记录
- 坑 1：[现象] → [原因] → [解决]
- 坑 2：...

## 总结
- 适用场景
- 局限性
- 下一步可以探索什么

## 参考
- [文章/文档名](url)
```

## 规则

- **先说结论再解释**（倒金字塔结构）
- 代码块必须标注语言，必须可运行（不写伪代码除非声明）
- 技术名词首次出现时解释，不假设读者知道
- 图 > 表 > 文字（能可视化就不要纯文字）
- 每个代码块上方说明"这段做什么"，下方说明"运行结果"
- 避免"众所周知"、"显而易见"
- 标题用"动词 + 具体内容"（"用 Docker 部署 Next.js" > "Docker 学习笔记"）

## SEO 建议

- 标题包含核心关键词
- 第一段出现关键词
- 用 h2/h3 结构化
- 文末加"相关推荐"（如果是系列）
