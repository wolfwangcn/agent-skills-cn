# 公众号写作 (WeChat Article)

基于公众号推荐算法、爆款标题公式和排版最佳实践，生成高质量文章。

---

## 平台机制理解

### 推荐算法（2024+）

公众号推荐流量已超过关注流量。分发机制：

```
推荐权重 = 内容质量分 × 互动率 × 账号权重 × 时效性
```

- **内容质量分**：原创度、完读率、图文并茂、排版规范
- **互动率**：点赞/在看/收藏/评论/转发（前 2 小时决定是否进入推荐池）
- **账号权重**：更新频率、历史数据、粉丝活跃度、违规记录
- **时效性**：新文章有 6-12 小时的冷启动窗口

## 指令

帮用户写一篇适合公众号发布的文章。严格按以下流程执行。

### 第一步：需求确认

必须确认：
1. **主题**：核心观点/信息是什么？
2. **受众画像**：谁看？（年龄、职业、痛点）
3. **文章类型**：
   - 干货教学型（方法论、工具推荐）
   - 故事叙事型（案例、经历、人物）
   - 观点输出型（行业评论、趋势判断）
   - 盘点清单型（Top N、对比、合集）
   - 情绪共鸣型（焦虑、认同、愤怒）
4. **字数要求**：短文(800-1500) / 标准(1500-3000) / 长文(3000+)
5. **有无素材**：用户提供的数据/案例/观点

### 第二步：标题设计（给 5 个备选）

使用以下爆款公式，每篇至少生成 5 个标题备选：

| 公式 | 结构 | 示例 |
|------|------|------|
| **数字+痛点** | N个[方法]让你[解决痛点] | "月薪3万的人都在用的5个工具" |
| **悬念反转** | [看似A]+（其实B） | "我劝你别学Python了（除非你看完这篇）" |
| **身份标签** | [群体]不会告诉你的[秘密] | "985毕业生不会告诉你的3个真相" |
| **对比冲突** | [A]用[短时间]完成[B]用[长时间]的工作 | "同事用1小时完成我3天的工作" |
| **权威背书** | [大厂/名人]+[稀缺内容] | "腾讯内部流出的PPT模板" |
| **时效性** | [时间]+[必须做的事] | "2024年最后一个月，这件事别再拖了" |
| **How型** | 如何[具体成果]+[不用付出大代价] | "如何不花钱学好英语" |

**标题评分标准**：
- ✅ 13-20字（太长截断，太短信息不足）
- ✅ 包含数字或具体名词
- ✅ 有信息增量（看标题就知道能学到什么）
- ✅ 有情绪驱动力（好奇、焦虑、认同）
- ❌ 不标题党（内容要 deliver 标题的承诺）

### 第三步：结构设计

使用 **SCQA 框架**（麦肯锡金字塔原理）：

```
S - Situation（情境）：读者熟悉的场景
C - Complication（冲突）：打破常规的问题
Q - Question（问题）：读者心中产生的疑问
A - Answer（答案）：本文给出的解决方案
```

**文章骨架**：

```
标题
│
├── Hook（开头 3 句话）
│   ├── 方式1：痛点场景（"你有没有遇到过..."）
│   ├── 方式2：数据冲击（"90%的人不知道..."）
│   ├── 方式3：故事开场（"上周，我的一个朋友..."）
│   └── 方式4：反常识（"你以为X，其实Y"）
│
├── 正文（3-5个核心点）
│   ├── ▎小标题1
│   │   └── 论点 + 论据/案例/数据 + 过渡句
│   ├── ▎小标题2
│   │   └── 论点 + 论据/案例/数据 + 过渡句
│   └── ▎小标题3
│       └── 论点 + 论据/案例/数据 + 过渡句
│
├── 金句/总结（1-2句，适合截图转发）
│
└── 结尾
    ├── 方式1：提问引导评论（"你觉得呢？"）
    ├── 方式2：行动呼吁（"今天就开始..."）
    └── 方式3：悬念预告（"下篇我会讲..."）
```

### 第四步：正文撰写

**情绪共鸣四驱动力**（驱动分享的核心）：
1. **焦虑**："不做X你就会被淘汰" → 驱动收藏
2. **好奇**："原来还有这种操作" → 驱动点击
3. **认同**："说出了我想说的话" → 驱动转发
4. **愤怒**："这也太离谱了吧" → 驱动评论

每段必须包含至少一个：
- 具体数据或统计
- 真实案例或故事
- 可操作的建议

### 第五步：排版输出

**排版规则**（基于移动端阅读体验）：

| 规则 | 原因 |
|------|------|
| 每段 2-4 行，绝不超过 5 行 | 手机屏幕小，密集文字劝退 |
| 段间空一行 | 呼吸感 |
| 重点**加粗**，每段最多 1 处 | 过多加粗等于没加粗 |
| 小标题用 ▎ 前缀或「」包裹 | 公众号不支持 Markdown |
| 每 300-500 字插入一张图 | 打断阅读疲劳 |
| emoji 点缀，每段 0-1 个 | 增加亲和力但不影响阅读 |
| 不用表格 | 手机端表格显示差 |
| 不用 # 标题语法 | 编辑器不支持 |
| 正文字号 15px，行距 1.75 | 最佳可读性 |
| 首行不缩进 | 移动端阅读习惯 |

**排版工具推荐**（告诉用户）：
- 135编辑器：https://www.135editor.com/
- 壹伴插件：https://yiban.io/

## 质量评判标准

输出后用以下 checklist 自查：

- [ ] 标题 13-20 字，含数字或具体名词？
- [ ] 开头 3 句话能抓住注意力？（不是"随着...的发展"）
- [ ] 每个论点有数据/案例支撑？（不是空洞观点）
- [ ] 读者能获得明确价值？（实用技巧/新认知/情绪共鸣）
- [ ] 有一句"金句"适合截图转发？
- [ ] 段落都在 2-4 行以内？
- [ ] 结尾有互动引导？
- [ ] 总字数符合要求？
- [ ] 口语化程度够？（朗读不觉得尴尬）

## 反面案例

❌ **开头**："随着人工智能的快速发展，越来越多的人开始关注AI..."（废话，没有任何信息增量）

✅ **开头**："上周我用 AI 写了一封辞职信，老板看完给我涨了薪。"（故事+悬念+反转）

❌ **段落**：连续 200 字没有换行，没有案例，全是抽象观点

✅ **段落**：2-3 行一个点 + 具体案例 + 数据佐证

❌ **结尾**："感谢阅读，希望对你有帮助！"

✅ **结尾**："你踩过哪些坑？评论区聊聊。"
