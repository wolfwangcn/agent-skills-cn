# 公众号写作 (WeChat Article)

按照微信公众号的阅读习惯和排版风格生成文章。

## 指令

帮用户写一篇适合微信公众号发布的文章。先确认主题和风格，再按下方结构输出。

## 写作前确认

1. **主题**：写什么？核心观点是什么？
2. **受众**：给谁看？（技术人 / 职场人 / 大众）
3. **风格**：干货型 / 故事型 / 观点型 / 盘点型
4. **字数**：短文(800-1500) / 中文(1500-3000) / 长文(3000+)

## 文章结构

```
标题（3 个备选）
│
├── 开头（3 句话内制造悬念/共鸣/冲突）
│
├── 正文
│   ├── ▎核心观点 1
│   │   └── 论证 + 案例/数据
│   ├── ▎核心观点 2
│   │   └── 论证 + 案例/数据
│   └── ▎核心观点 3
│       └── 论证 + 案例/数据
│
├── 金句（一句话总结，适合截图转发）
│
└── 结尾（引导互动：提问 / 投票 / "你怎么看？"）
```

## 排版规则

- 段落短：每段 2-4 行，绝不超过 5 行
- 多用空行分隔，留白 > 密集
- 重点**加粗**，但一段最多加粗一处
- 小标题用 ▎ 前缀或「」包裹
- 适当 emoji 点缀（每段 0-1 个）
- **不用 Markdown 标题语法**（公众号编辑器不支持 # ##）
- **不用表格**（手机端显示差）

## 规则

- 开头 3 行决定读者留不留 — 禁止"随着...的发展"式开场
- 一个段落只讲一件事
- 口语化 > 书面化（"说白了" > "换言之"）
- 结尾禁止"感谢阅读"、"希望对你有帮助"
- 标题不标题党，但要有信息增量和点击欲
- 给出 3 个标题备选 + 推荐理由

## 反面案例

❌ "随着人工智能的快速发展，越来越多的人开始关注..."（废话开头）
❌ 一段 200 字没有换行（手机阅读体验差）
❌ 全文没有一个具体案例或数据（空洞）
✅ "昨天我被 AI 坑了一把。" （故事开头，有悬念）
