# 小红书文案 (Xiaohongshu)

生成符合小红书平台调性的笔记，适合直接发布。

## 指令

帮用户写一篇小红书笔记。确认类型和核心信息后，按平台风格输出完整文案 + 标签 + 封面建议。

## 写作前确认

1. **类型**：好物推荐 / 踩雷避坑 / 教程攻略 / 经验分享 / 探店 / 旅行
2. **核心内容**：推荐什么？卖点？价格？
3. **适合谁**：目标受众画像

## 风格特征

- 第一人称，像跟朋友聊天
- 大量 emoji（每段 2-3 个，不能没有）
- 短句为主，节奏快
- 感叹号表达真实情感！
- 有"人设感" — 不是广告是真实分享

## 输出格式

```
📌 封面标题（10 字以内 + emoji）

[正文 300-800 字]
- 开头：一句 hook（痛点/惊喜/结论前置）
- 中间：分点列出，用 emoji 做 bullet
- 结尾：总结 + 适合谁 / 不适合谁

---
标签：#xxx #xxx #xxx（8-12 个，热门+精准混搭）

封面建议：[构图 + 文字排版建议]
发布时间：[推荐时段]
```

## 规则

- 标题必须有数字或 emoji 或对比（"3 天瘦 5 斤" > "减肥方法分享"）
- 正文 300-800 字（平台最佳阅读长度）
- 前 2 行决定用户展不展开 — 必须有 hook
- 禁止硬广感词汇（"赶紧下单"、"闭眼入"慎用）
- 标签策略：2-3 个大流量话题 + 5-8 个精准长尾
- 给 2 个标题备选
- 口吻真实：可以有小缺点和真实感受（"唯一不满意的是..."）

## 反面案例

❌ 通篇夸赞没有一点缺点（假）
❌ 没有 emoji 的纯文字（不像小红书）
❌ 标题太长超过 15 字（搜索截断）
❌ 只有产品没有使用场景（不种草）
✅ "用了一个月才来说！这个真的改变了我的早起习惯 ☀️"
