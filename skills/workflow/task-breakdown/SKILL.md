# 任务拆解 (Task Breakdown)

## 描述
把复杂任务拆解为可执行的小步骤，每步有明确的完成标准。防止 AI 一口气做太多、做错方向。

## 触发条件
当任务包含多个步骤或预计耗时较长时激活。

## 流程

### 第一步：评估复杂度
- 这个任务能一步完成吗？→ 能就直接做
- 需要多步？→ 进入拆解模式

### 第二步：拆解
将任务拆为 3-7 个步骤，每步满足：
1. **原子性** — 一步做一件事
2. **可验证** — 做完能检查对不对
3. **有顺序** — 标明依赖关系

### 第三步：输出格式

```markdown
## 任务：[总任务描述]

预计步骤：X 步

- [ ] 步骤 1：[描述] → 验证：[怎么确认完成]
- [ ] 步骤 2：[描述] → 验证：[怎么确认完成]
- [ ] 步骤 3：[描述] → 验证：[怎么确认完成]

风险/注意：[可能出问题的地方]
```

### 第四步：逐步执行
- 每完成一步，报告进度
- 遇到问题立即暂停，不要跳步
- 允许用户随时调整计划

## 规则
- 步骤不超过 7 个（超过就再拆子任务）
- 每步用动词开头（"创建"、"配置"、"测试"）
- 先做最可能失败的步骤（fail fast）
- 预估时间：给用户心理预期
