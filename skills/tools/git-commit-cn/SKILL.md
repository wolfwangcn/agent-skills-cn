# Git 提交规范 (Git Commit CN)

## 描述
规范化中文 Git commit message，保持项目历史清晰可读。

## 触发条件
执行 git commit 时自动应用此规范。

## 格式

```
<类型>(<范围>): <简述>

[可选正文]

[可选脚注]
```

## 类型

| 类型 | 说明 | 示例 |
|------|------|------|
| feat | 新功能 | feat(用户): 添加手机号登录 |
| fix | 修复 | fix(支付): 修复重复扣款问题 |
| docs | 文档 | docs: 更新 API 说明 |
| style | 格式 | style: 统一缩进为 2 空格 |
| refactor | 重构 | refactor(订单): 拆分订单服务 |
| perf | 性能 | perf: 优化列表加载速度 |
| test | 测试 | test(auth): 补充登录测试用例 |
| chore | 杂项 | chore: 升级依赖版本 |

## 规则
- 简述不超过 50 字
- 用中文写，类型前缀用英文
- 简述用动词开头（"添加"、"修复"、"优化"）
- 不加句号
- 正文和简述之间空一行
- 一个 commit 只做一件事

## 示例

```
feat(文章): 添加 Markdown 导出功能

支持将文章导出为 .md 文件，包含图片本地化处理。

closes #42
```

```
fix(登录): 修复验证码倒计时重置问题

原因：组件卸载时未清除定时器
```
