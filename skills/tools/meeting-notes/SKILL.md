# 会议纪要 (Meeting Notes)

从对话、语音转文字或会议记录中提取结构化纪要，突出决策和行动项。

## 指令

分析用户提供的会议内容，提取核心信息，按结构输出。行动项是最重要的部分 — 必须有负责人和截止时间。

## 输出格式

```markdown
# 会议纪要

**日期**：YYYY-MM-DD HH:MM
**参会人**：[人员列表]
**主题**：[一句话主题]
**时长**：约 X 分钟

---

## 📌 核心决策
1. [决定了什么] — 原因：[为什么这么决定]
2. [决定了什么] — 原因：[为什么]

## ✅ 行动项 (Action Items)
| # | 事项 | 负责人 | 截止日期 | 备注 |
|---|------|--------|---------|------|
| 1 | [具体任务] | @谁 | MM-DD | |
| 2 | [具体任务] | @谁 | MM-DD | |

## 💬 讨论要点

### 话题 1：[标题]
- [要点]
- [不同意见]
- **结论**：[达成的共识]

### 话题 2：[标题]
- [要点]
- **结论**：[达成的共识 / 未达共识]

## ❓ 待定
- [需要后续确认的问题] → 谁去确认？何时？

## 📅 后续
- 下次会议：[时间，如果确定了]
- 跟进节点：[什么时候检查行动项进展]
```

## 规则

- **行动项是灵魂** — 没有负责人和截止时间的行动项等于没有
- 区分"决定"和"讨论" — 只有达成共识的才是决定
- 用人名不用"某位同事"、"有人提到"
- 如果原文模糊，标注 [待确认]
- 纪要长度 ≤ 原文 1/3
- 不加入会议中没提到的内容
- 讨论中的分歧也要记录（"A 认为...，B 认为..."）
- 输出后问："需要我把行动项发给相关人吗？"

## 处理技巧

- 语音转文字质量差？先做纠错再整理
- 内容太长？先问用户关注哪些话题
- 多个会议混在一起？拆分后分别整理
